# commit 保存代码规范 type：用于说明 commit 的类别，规定为如下几种

1. feat：新增功能；
2. fix：修复 bug；
3. docs：修改文档；
4. refactor：代码重构，未新增任何功能和修复任何 bug；
5. build：改变构建流程，新增依赖库、工具等（例如 webpack 修改）；
6. style：仅仅修改了空格、缩进等，不改变代码逻辑；
7. perf：改善性能和体现的修改；
8. chore：非 src 和 test 的修改；
9. test：测试用例的修改；
10. ci：自动化流程配置修改；
11. revert：回滚到上一个版本；
